Coding style:
	
	- använda $_ prefix på input variabler som är user supplied till en funktion
	- använda foreach() istället för for() för att iterera över sql resultset arrayer i den mån det är praktiskt
  	(=mer lättläst kod,kanske lite prestanda)

	- Strings named $q always hold SQL queries
	- functions curly braces always on separate lines. use separate lines in other cases too where apropriate (for code readability)


File format:
	- For text files, always use UTF-8 encoding and Unix line ending (LF)


Subversion rules/common sense:
	
	- Never checkin non-functional code
		Always run/compile your code before a check-in to ensure that it dont produce parse errors/compile errors,
		in such state as it will break the HEAD codebase for other developers.
		When you realize that you did, please commit fix as soon as you possibly can.


Naming rules:

	Because the code base is cross-languages, often mixed in content, I decided to use the following
	coding style to make it easier to distinguish code:
		
	For PHP code: functionNamingLooksLikeThis(), andMaybeThis(), andSomethingElse()
		
	For Javascript code: function_naming_looks_like_this()

	For XHTML div tag names: container_name, some_object_header
		
	For SQL queries (keywords in caps): SELECT * FROM tblUsers WHERE userId=4
